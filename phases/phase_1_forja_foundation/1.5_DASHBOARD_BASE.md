# 1.5 DASHBOARD BASE

## PURPOSE
Define the minimum operational dashboard foundation for FORJA.

## OBJECTIVES
- Show real backend state, not simulated readiness.
- Provide a clean baseline for future operational dashboards.
- Keep the dashboard useful before adding analytics.

## REQUIRED COMPONENTS
- Backend online/offline display.
- `/health` response visibility.
- System version display.
- Timestamp or last refresh indicator.
- Clear empty/error states.

## RULES
- Do not create fake charts, fake analytics, or mock operational metrics.
- Do not add realtime behavior until the backend supports it.
- Dashboard must expose truth, not marketing polish.

## ECOSYSTEM COMPATIBILITY
- Dashboard patterns must be reusable for ecosystem status surfaces.
- Health status must align with backend contracts.

## SUCCESS CONDITIONS
- Dashboard base is operationally honest.
- Users can see whether FORJA backend is reachable.
- Future metrics can be added without redesign.
