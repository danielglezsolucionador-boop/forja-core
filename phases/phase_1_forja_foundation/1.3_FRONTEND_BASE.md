# 1.3 FRONTEND BASE

## PURPOSE
Define the frontend foundation for FORJA using React and Tailwind as a reusable application shell.

## OBJECTIVES
- Establish a clean frontend structure for dashboards and operational pages.
- Define shared component boundaries.
- Keep the UI functional and real, without fake metrics or placeholder product claims.

## REQUIRED COMPONENTS
- React application base.
- Tailwind styling baseline.
- Shared component structure.
- Environment-aware API URL handling.
- Basic operational layout conventions.

## RULES
- Do not create enterprise dashboards with fake data.
- Do not add charts, realtime behavior, or advanced UI unless required by a phase.
- Frontend must consume real backend contracts when implemented.

## ECOSYSTEM COMPATIBILITY
- Shared UI patterns must be reusable by Hermes, Centinela, Cerebro, and future apps.
- The frontend base must not depend on a single product workflow.

## SUCCESS CONDITIONS
- Frontend base is documented as reusable infrastructure.
- Component boundaries are clear.
- UI work remains tied to real operational data.
