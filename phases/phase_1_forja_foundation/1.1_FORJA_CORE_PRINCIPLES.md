# 1.1 FORJA CORE PRINCIPLES

## PURPOSE
Define the non-negotiable operating principles for FORJA as the technology factory of the ecosystem.

## OBJECTIVES
- Establish FORJA as reusable infrastructure, not a magic builder or temporary MVP.
- Keep every future module incremental, verifiable, and reversible.
- Prioritize stable foundations before automation or generation.

## REQUIRED COMPONENTS
- Reusable-first architecture principles.
- Governance-first execution rules.
- Operational validation expectations.
- Modular monorepo orientation.

## RULES
- Do not create fake AI, fake readiness, or demo-only systems.
- Do not add features outside the authorized phase.
- Read existing context before creating or changing artifacts.
- Prefer small, isolated changes with clear validation.

## ECOSYSTEM COMPATIBILITY
- FORJA must support Hermes, Cerebro, Centinela, API Hub, and future client apps through reusable technical foundations.
- FORJA must not replace operational ownership of other ecosystem apps.

## SUCCESS CONDITIONS
- FORJA has a clear identity as infrastructure factory.
- All phases can be executed without inventing architecture.
- Future work can be audited against these principles.
