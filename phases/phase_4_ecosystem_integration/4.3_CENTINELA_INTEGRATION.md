# 4.3 CENTINELA INTEGRATION

## PURPOSE
Define FORJA compatibility with Centinela security and audit expectations.

## OBJECTIVES
- Ensure generated outputs can be audited.
- Prepare security review hooks for future phases.
- Keep unsafe generation blocked by governance.

## REQUIRED COMPONENTS
- Audit request concept.
- Security review metadata.
- Risk classification boundary.
- Blocker reporting.
- Approval trace compatibility.

## RULES
- Do not bypass security review for generated outputs.
- Do not create fake security pass states.
- Do not expose secrets in documentation or generated metadata.

## ECOSYSTEM COMPATIBILITY
- Centinela should be able to inspect FORJA output metadata later.
- Security escalation must remain human-readable.

## SUCCESS CONDITIONS
- Audit compatibility is documented.
- Risk and blocker handling are explicit.
- Security is built into the process, not patched after.
