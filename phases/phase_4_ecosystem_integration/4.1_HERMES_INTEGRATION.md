# 4.1 HERMES INTEGRATION

## PURPOSE
Define how FORJA should integrate with Hermes without modifying Hermes directly.

## OBJECTIVES
- Identify safe integration boundaries.
- Preserve Hermes runtime ownership.
- Support future generated tools that can report or consume operational context.

## REQUIRED COMPONENTS
- Read-only integration concept.
- API contract discovery boundary.
- Dashboard compatibility notes.
- Governance handoff concept.
- No-runtime-touch rule.

## RULES
- Do not modify Hermes runtime, polling, provider bridge, execution, or dashboard from FORJA documentation.
- Do not assume Hermes internals are FORJA modules.
- Integration must be opt-in and reversible.

## ECOSYSTEM COMPATIBILITY
- FORJA can create tools compatible with Hermes contracts.
- Hermes remains the operational runtime system, not a factory output.

## SUCCESS CONDITIONS
- Hermes boundary is protected.
- Future integration can happen through explicit contracts.
- No accidental dependency is introduced.
