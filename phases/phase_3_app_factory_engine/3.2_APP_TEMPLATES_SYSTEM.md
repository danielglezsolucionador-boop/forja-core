# 3.2 APP TEMPLATES SYSTEM

## PURPOSE
Define the template system for reusable FORJA application outputs.

## OBJECTIVES
- Identify template categories without creating implementation files.
- Keep templates realistic and tied to real app needs.
- Preserve compatibility with shared modules.

## REQUIRED COMPONENTS
- Landing page template category.
- Dashboard template category.
- Admin panel template category.
- SaaS base template category.
- API service template category.

## RULES
- Do not create fake visual templates.
- Do not include business claims or sample data as real output.
- Templates must be documented as patterns, not generated products yet.

## ECOSYSTEM COMPATIBILITY
- Templates must support React + Tailwind frontend and FastAPI backend conventions.
- Templates should be adaptable for ecosystem apps and client deliverables.

## SUCCESS CONDITIONS
- Template categories are explicit.
- Shared module dependencies are clear.
- Future template implementation can be validated.
