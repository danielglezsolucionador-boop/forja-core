# 3.6 FACTORY OUTPUT VALIDATION

## PURPOSE
Define validation rules for future FORJA-generated outputs.

## OBJECTIVES
- Ensure generated apps are structurally complete.
- Detect missing files, broken imports, and invalid contracts.
- Prevent fake readiness after scaffolding.

## REQUIRED COMPONENTS
- File completeness checklist.
- Dependency validation concept.
- Endpoint contract checks.
- Frontend render checks.
- Governance and audit checks.

## RULES
- Do not mark output ready until validation passes.
- Do not ignore warnings that affect startup or user workflows.
- Do not use mocks as proof of production readiness.

## ECOSYSTEM COMPATIBILITY
- Validation should work for internal apps and client deliverables.
- Output must respect shared modules and standards.

## SUCCESS CONDITIONS
- Factory output validation is explicit.
- Readiness depends on evidence.
- Future generated apps can be handed off safely.
