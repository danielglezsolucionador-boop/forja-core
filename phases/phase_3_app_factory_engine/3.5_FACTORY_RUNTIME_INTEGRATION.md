# 3.5 FACTORY RUNTIME INTEGRATION

## PURPOSE
Define how future factory workflows should integrate with operational runtime surfaces.

## OBJECTIVES
- Keep factory status observable.
- Separate runtime status from generation logic.
- Support safe progress tracking and failure reporting.

## REQUIRED COMPONENTS
- Factory job status concept.
- Runtime event emission concept.
- Error and blocker reporting.
- Audit-friendly execution metadata.
- Dashboard compatibility notes.

## RULES
- Do not build runtime systems in documentation phase.
- Do not hide failed generation steps.
- Runtime integration must be status-only until authorized.

## ECOSYSTEM COMPATIBILITY
- Runtime signals should support dashboards and governance tools.
- Integration must not modify Hermes runtime or other existing systems.

## SUCCESS CONDITIONS
- Runtime integration scope is documented.
- Factory progress can be made observable later.
- No existing runtime architecture is changed.
