# 3.1 APP FACTORY CORE

## PURPOSE
Define the core app factory model for FORJA without implementing generation yet.

## OBJECTIVES
- Describe what FORJA can create and what remains manual.
- Establish controlled factory inputs and outputs.
- Keep the factory practical, not magical.

## REQUIRED COMPONENTS
- App type taxonomy.
- Input specification concept.
- Output package boundary.
- Validation gate concept.
- Reusable module selection rules.

## RULES
- Do not call the factory autonomous unless it can be validated.
- Do not generate real apps in documentation phase.
- Every future factory action must be reversible and auditable.

## ECOSYSTEM COMPATIBILITY
- Factory outputs must align with ecosystem standards and shared modules.
- Generated structures must be usable by internal and client-facing systems.

## SUCCESS CONDITIONS
- Factory scope is clear.
- Outputs are bounded.
- Future implementation can follow documented gates.
