# 2.2 SHARED FRONTEND MODULES

## PURPOSE
Define reusable frontend modules for dashboards, admin panels, and operational tools.

## OBJECTIVES
- Standardize common UI primitives.
- Keep components reusable without fake data.
- Support fast creation of real operational interfaces.

## REQUIRED MODULES
- Layout shell.
- Status card.
- Data table boundary.
- Empty/error/loading states.
- API status indicator.

## RULES
- Do not bind shared components to one backend response shape unless documented.
- Do not include fake charts or static operational claims.
- Shared components must accept real data through explicit props.

## ECOSYSTEM COMPATIBILITY
- UI modules must support dashboards for FORJA, Hermes, Centinela, and future apps.
- Styling must remain compatible with React and Tailwind.

## SUCCESS CONDITIONS
- Reusable UI boundaries are clear.
- Components can be composed without rewriting.
- Operational truth is preserved in the UI.
