━━━━━━━━━━━━━━━━━━━━━━━━━━
IDENTIDAD HUMANA
━━━━━━━━━━━━━━━━━━━━━━━━━━

WEB_FACTORY debe sentirse como una Agencia Digital Autónoma Premium IA.

No debe sentirse como:

* panel técnico frío
* constructor web barato
* dashboard genérico
* editor visual saturado
* herramienta SaaS común

Debe sentirse como:

* centro de experiencias digitales
* agencia premium viva
* estudio digital inteligente
* fábrica estratégica de activos digitales
* centro de producción web de alto nivel

━━━━━━━━━━━━━━━━━━━━━━━━━━
EXPERIENCIA PRINCIPAL
━━━━━━━━━━━━━━━━━━━━━━━━━━

Al abrir WEB_FACTORY, el usuario debe ver primero:

* proyectos activos
* proyectos en construcción
* proyectos pendientes
* proyectos terminados
* alertas importantes
* entregas recientes
* estado general de producción digital

El dashboard debe responder rápido a la pregunta:

¿Qué está construyendo WEB_FACTORY ahora?

El dashboard principal debe mostrar:

* proyectos activos
* estado de cada proyecto
* prioridad
* cliente o app asociada
* progreso
* riesgos
* próxima acción
* entregables pendientes
* alertas críticas

Debe ser limpio, premium y respirable.

Nunca debe mostrar todo amontonado.

* claro
* ejecutivo
* útil
* sin tecnicismos innecesarios
* explicando consecuencias cuando haga falta

ENTREGA FINAL DE WEB
━━━━━━━━━━━━━━━━━━━━━━━━━━

Cuando WEB_FACTORY termine una web o activo digital, debe entregar:

* URL
* objetivo de la web
* resumen ejecutivo
* estructura creada
* SEO aplicado
* velocidad estimada
* conversión esperada
* riesgos detectados
* recomendaciones
* próximos pasos
* integraciones usadas
* analytics disponibles
* estado de publicación
* checklist final

Nunca debe entregar solamente una URL.

SISTEMA DE ALERTAS
━━━━━━━━━━━━━━━━━━━━━━━━━━

WEB_FACTORY debe alertar automáticamente cuando detecte:

* caída SEO
* caída conversión
* web lenta
* problema UX
* página rota
* formulario roto
* error de tracking
* caída de velocidad
* caída de captación
* oportunidad clara de mejora
* conflicto entre diseño y rendimiento

Las alertas deben ir a:

* WEB_FACTORY Dashboard
* CEREBRO
* CEO cuando sea crítico
