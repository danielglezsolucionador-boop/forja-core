## Human Experience Identity

Centinela must feel like a premium operational security command system: calm, precise, defensive, evidence-based, and ready for crisis without becoming noisy.

## What Must Be Preserved

- CENTINELA AI SECURITY identity.
- Existing dark command-center feeling.
- Current security vocabulary: threats, prompts, incidents, runtime, governance, forensics, policies.
- Timeline and threat table patterns.
- Executive and technical modes.

## Human Cabin Principles

- Executive signal first.
- Technical detail available on demand.
- No fake urgency.
- No decorative metrics.
- No alert saturation.
- No cyberpunk noise.
- No hidden uncertainty.

## Required Views

1. Executive Security State
   - security health score,
   - active freeze state,
   - highest risk,
   - affected systems,
   - required decision.

2. Threat Intelligence
   - active threats,
   - blocked threats,
   - uncertain findings,
   - evidence confidence.

3. Metric Provenance
   - real,
   - simulated,
   - stale,
   - unknown,
   - last updated,
   - source.

4. Crisis Focus Mode
   - only critical threats,
   - containment path,
   - rollback/freeze state,
   - CEO approval requirement,
   - operational impact.

5. Governance Visibility
   - policy applied,
   - decision reason,
   - actor,
   - timestamp,
   - response target.

## Mobile Rules

Mobile must not be a compressed dashboard. It must show:

- current security posture,
- critical threat if any,
- action needed,
- freeze status,
- last trusted update.

## Cognitive Safety Rules

Centinela must reduce noise under pressure. If many alerts exist, it should group by impact and show the smallest useful set first.
