# PLUMA — HUMAN CABIN PROFILE

## PROPÓSITO HUMANO

Pluma NO debe sentirse como:
- un chatbot IA genérico
- un CMS aburrido
- una fábrica automática de textos
- una herramienta fría de marketing

Debe sentirse como:

un estudio editorial premium, inteligente, elegante y creativo diseñado para producir contenido de altísimo nivel sin generar caos, fatiga ni ruido operacional.

━━━━━━━━━━━━━━━━━━━

# OBJETIVO HUMANO

Permitir que:

- CEO
- Cerebro
- Marketing
- Marca Personal

puedan:
- investigar
- crear
- revisar
- reutilizar
- transformar
- organizar
- descargar
- aprobar

contenido premium de forma natural, rápida y elegante.

━━━━━━━━━━━━━━━━━━━

La experiencia debe sentirse:

- premium
- limpia
- elegante
- enfocada
- creativa
- respirable
- moderna
- silenciosa visualmente
- inteligente

Inspiración visual:

- Apple
- Notion premium
- Arc Browser

━━━━━━━━━━━━━━━━━━━

El usuario debe sentir:

- claridad mental
- enfoque creativo
- control editorial
- orden intelectual
- velocidad operacional
- inspiración
- tranquilidad visual

NO:
- estrés visual
- dashboards llenos
- ruido
- fatiga cognitiva
- paneles inútiles

━━━━━━━━━━━━━━━━━━━

# INTERACCIÓN CON PLUMA

Pluma debe comunicarse:

- claramente
- naturalmente
- profesionalmente
- elegantemente
- inteligentemente

NO:
- robóticamente
- exageradamente técnica
- artificialmente emocional
- saturada de texto

━━━━━━━━━━━━━━━━━━━

# ESTILO CONVERSACIONAL

Pluma debe:

- explicar bien
- resumir inteligentemente
- contextualizar
- sugerir formatos útiles
- sugerir mejoras editoriales
- sugerir reutilización contenido

Debe sentirse:

como un editor estratégico premium.

- convertir formatos

# EXPERIENCIA DE ESCRITURA

El editor debe sentirse:

- limpio
- premium
- silencioso
- elegante
- cómodo para escribir largo tiempo

Debe priorizar:
- lectura
- claridad
- foco creativo
- navegación rápida

━━━━━━━━━━━━━━━━━━━

# INVESTIGACIONES PROFUNDAS

Cuando Pluma entregue investigaciones:

debe mostrarlas:
- organizadamente
- claramente
- visualmente respirables
- fáciles de explorar

Debe separar:
- hallazgos
- riesgos
- oportunidades
- tendencias
- conclusiones
- recomendaciones

━━━━━━━━━━━━━━━━━━━

# ENTREGA DE CONTENIDO

Pluma debe poder entregar:

- PDF
- DOCX
- EPUB
- markdown
- drafts LinkedIn
- drafts Twitter/X
- artículos completos
- papers
- ebooks
- investigaciones

Todo descargable:
desde móvil o dashboard.

━━━━━━━━━━━━━━━━━━━

# EXPERIENCIA MÓVIL

PRIORIDAD ALTA.

Desde celular debe poder:

- leer investigaciones
- descargar PDFs
- revisar drafts
- aprobar publicaciones
- buscar contenido viejo
- pedir investigaciones nuevas
- revisar libros
- revisar papers

Sin experiencia incómoda.

━━━━━━━━━━━━━━━━━━━

# ALERTAS

Pluma debe alertar SOLO cuando:

- una investigación terminó
- un libro terminó
- un paper terminó
- existe oportunidad importante
- existe degradación reputacional
- existe problema editorial importante

NO:
spam operacional.

━━━━━━━━━━━━━━━━━━━

- exceso de notificaciones
- exceso de paneles
- exceso de colores
- exceso de métricas

Priorizar:

- foco
- claridad
- respiración visual
- navegación intuitiva
