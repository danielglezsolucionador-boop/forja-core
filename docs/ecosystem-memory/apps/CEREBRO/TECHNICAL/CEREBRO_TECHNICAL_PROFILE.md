# CEREBRO — CABINA TÉCNICA

## 1. IDENTIDAD TÉCNICA

Cerebro debe ser un orquestador modular ejecutivo.

No debe ser una app monolítica gigante.

Debe estar diseñado para coordinar, decidir, registrar, aprender y delegar dentro del ecosistema IA.

---

## 2. STACK BASE RECOMENDADO

Backend principal:
Python

Framework:
FastAPI

Base de datos:
PostgreSQL

Memoria semántica / búsqueda:
Vector store futuro

Frontend:
React + Vite + Tailwind

Canal operativo:
Telegram

---

## 3. ARQUITECTURA GENERAL

Estructura conceptual:

---

## 4. PRINCIPIO CENTRAL

Cerebro debe funcionar por eventos.

No debe depender de loops infinitos.

Ejemplos de eventos:

- AIC detecta problema
- Centinela detecta riesgo
- FORJA termina fase
- Investigador detecta oportunidad
- Pluma genera análisis
- Hermes crea API
- CEO solicita decisión

---

## 5. MÓDULOS PRINCIPALES

core:
núcleo central de decisión.

nuclei:
núcleos ejecutivos especializados.

memory:
memoria corta, estratégica, histórica y aprendizaje.

telegram:
comunicación con CEO.

orchestration:
coordinación de apps.

validation:
revisión antes de actuar.

decision:
síntesis y toma de decisiones.

meetings:
reunión ejecutiva diaria.

ecosystem:
estado general de todas las apps.

logs:
trazabilidad.

learning:
aprendizaje continuo.

finance:
rentabilidad, costos, ingresos, ROI.

monitoring:
estado operacional.

---

## 6. MEMORIA

Cerebro debe tener 4 memorias:

1. Memoria corta:
estado actual del ecosistema.

2. Memoria estratégica:
objetivos, decisiones, roadmap, prioridades.

3. Memoria histórica:
qué pasó, qué falló, qué funcionó.

4. Memoria de aprendizaje:
patrones, preferencias del CEO, errores repetidos, mejoras.

---

## 7. LOGS EJECUTIVOS

Toda decisión importante debe registrar:

- qué ocurrió
- quién lo reportó
- qué núcleo recomendó
- qué decidió Cerebro
- por qué decidió eso
- impacto económico
- riesgo
- si fue reversible
- si requirió aprobación humana

---

## 8. TELEGRAM

Telegram será el canal principal con el CEO.

Debe soportar:

- alertas
- reportes
- preguntas
- aprobaciones
- resumen diario
- interrupciones críticas

Niveles:

1. silencioso
2. informativo
3. prioritario
4. crítico

---

## 9. EVENTOS

Todo debe convertirse en eventos internos.

Ejemplo:

EVENT:
AIC_REPORT_READY

EVENT:
CENTINELA_CRITICAL_RISK

EVENT:
FORJA_PHASE_COMPLETED

EVENT:
CEO_APPROVAL_REQUIRED

EVENT:
DAILY_EXECUTIVE_SUMMARY

---

## 10. REGLA DE SEGURIDAD

Si una acción es irreversible o puede causar daño económico, técnico o estratégico:

Cerebro debe pedir aprobación humana.

---

## 11. INTEGRACIONES

Cerebro debe integrarse con:

- AIC
- FORJA
- CENTINELA
- HERMES
- PLUMA
- INVESTIGADOR
- CENTRO DE DIRECCIÓN

Pero no debe reemplazarlos.

---

## 12. CENTRO DE DIRECCIÓN

Centro de Dirección no es Cerebro.

Centro de Dirección es el cockpit visual.

Cerebro decide y coordina.
Centro de Dirección muestra, organiza y visualiza.

---

## 13. CABINA TÉCNICA OBJETIVO

La cabina técnica debe permitir:

- ver estado de apps
- ver eventos
- ver decisiones
- ver riesgos
- ver tareas delegadas
- ver bloqueos
- ver costos
- ver progreso
- ver alertas
- ver auditorías
- ver historial

---

## 14. PRINCIPIO DE ESCALABILIDAD

Cada módulo debe poder evolucionar sin romper todo Cerebro.

Cada núcleo debe poder ser auditado, reemplazado o mejorado de forma independiente.

---

## 15. PRINCIPIO DE NO ALUCINACIÓN

Cerebro no debe declarar hechos técnicos sin evidencia.

Debe usar:

- logs
- eventos
- respuestas API
- reportes de apps
- auditorías
- memoria registrada

Si no hay evidencia:

NO CONFIRMADO.
REQUIERE VALIDACIÓN.

---

## 16. RESULTADO ESPERADO

Cerebro debe convertirse en el sistema técnico central de coordinación ejecutiva del ecosistema IA.

Debe permitir que el ecosistema trabaje incluso cuando el CEO no esté frente a la computadora, manteniendo trazabilidad, control, límites y decisiones supervisadas.
